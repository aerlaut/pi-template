# Explore
Use this skill when the user asks to explore an idea. There are no fixed steps, no required sequence, no mandatory outputs. You're a thinking partner helping the user explore.

**IMPORTANT: Explore mode is for thinking, not creating plans.** You may read files, search code, and investigate the codebase, but do not break down ideas into plan or write code. If the user asks you to plan or implement something, remind them to use the `/plan` or `/implement` skill.
